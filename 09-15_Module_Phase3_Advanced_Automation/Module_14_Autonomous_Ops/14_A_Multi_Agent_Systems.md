📘 Module 14 — Autonomous Network Operations Framework (Section A)

📌 S — Scenario 1 (Non-Banking)

একটা Restaurant Kitchen-এ একজন মানুষ একাই Order নেওয়া, রান্না করা, Plate সাজানো, এবং Serve করা — সবকিছু করলে কাজ ধীর হয়ে যায় এবং ভুল বেশি হয়। এর বদলে আলাদা আলাদা মানুষ আলাদা আলাদা কাজ করে (একজন Order নেয়, একজন রান্না করে, একজন Serve করে), এবং তারা একে অপরের সাথে সমন্বয় করে কাজ করে।

📌 S — Scenario 2 (Banking, Non-Technical)

Nord Bank-এর একটা Automation-কে যদি একাই সব কাজ করতে বলা হয় — Log Monitor করা, বিশ্লেষণ করা, Report লেখা, এবং Notify করা — তাহলে এই একটা মাত্র Automation অনেক জটিল হয়ে যায় এবং একটা অংশে সমস্যা হলে পুরোটাই আটকে যায়।

🎯 T — Task

আজকের লক্ষ্য:
- Multi-Agent System বলতে কী বোঝায় তা বোঝা — একটা কাজ একাধিক ছোট Agent-এর মধ্যে ভাগ করে দেওয়ার ধারণা
- একটা মাত্র বড়, সবকিছু করা Agent-এর বদলে একাধিক ছোট, নির্দিষ্ট-কাজের Agent ব্যবহারের সুবিধা বোঝা
- Agent-দের মধ্যে সমন্বয় (Coordination) বলতে কী বোঝায় তার প্রাথমিক ধারণা পাওয়া

👀 O — Output

Module শেষে learner বলতে পারবে:
- Single Agent আর Multi-Agent System-এর মধ্যে মূল পার্থক্য কী
- কেন একটা কাজকে ছোট ছোট, নির্দিষ্ট দায়িত্বে ভাগ করা সুবিধাজনক
- Agent-রা কীভাবে একে অপরের সাথে তথ্য আদান-প্রদান করে কাজ সম্পন্ন করতে পারে, সাধারণভাবে

🤔 R — Reason

একটা মাত্র Agent যদি সবকিছু করার চেষ্টা করে, তাহলে সেটা জটিল হয়ে যায় এবং Debug করা কঠিন হয়ে যায় — কোন অংশে সমস্যা হয়েছে তা বোঝা কঠিন। যদি প্রতিটা Agent একটা নির্দিষ্ট, ছোট দায়িত্ব পালন করে (যেমন একটা শুধু Monitor করে, আরেকটা শুধু Analyze করে), তাহলে প্রতিটা অংশ আলাদাভাবে বোঝা, Test করা, এবং প্রয়োজনে পরিবর্তন করা সহজ হয়।

🧠 Memory Tip

Multi-Agent System-কে ভাবা যেতে পারে একটা "Restaurant Kitchen Team"-এর মতো — প্রতিটা Station-এর একজন করে দায়িত্বে থাকে, কেউ একা সব কাজ করে না, কিন্তু সবাই মিলে একটা Order সম্পন্ন করে।

⚠️ L — Limitation

- একাধিক Agent ব্যবহার করলে তাদের মধ্যে সমন্বয় (Coordination) ঠিকভাবে না হলে, একটা Agent-এর ভুল আরেকটাতে ছড়িয়ে যেতে পারে
- Agent সংখ্যা বাড়লে পুরো System বোঝা এবং Manage করা জটিল হয়ে যায় — Single Agent-এর তুলনায় Setup করা কঠিন
- একটা Agent যদি অন্য Agent-এর কাছ থেকে ভুল বা Incomplete তথ্য পায়, তাহলে সে ভুল সিদ্ধান্ত নিতে পারে, এবং এই ভুল Chain-এর পরের Agent-এও ছড়িয়ে যেতে পারে
- সব কাজের জন্য Multi-Agent দরকার হয় না — সহজ, সরাসরি কাজে একটা Single Agent-ই যথেষ্ট এবং সহজ হতে পারে

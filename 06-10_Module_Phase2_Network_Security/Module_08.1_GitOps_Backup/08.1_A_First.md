📘 Module 08.1 — First Assignment: Config Backup Basics (Section A)

📌 S — Scenario 1 (Non-Banking)

একজন Freelance Web Developer একটা Client-এর Website-এর Setting নিয়মিত হাতে Copy করে নিজের Computer-এ রাখে, যাতে ভুল কিছু হলে আগের Version-এ ফিরে যাওয়া যায়। কিন্তু কাজ ব্যস্ত থাকলে সে মাঝে মাঝে এই কাজ ভুলে যায়, এবং একবার Website Crash করার পর দেখা যায় সবচেয়ে সাম্প্রতিক Copy টা ২ মাস আগের।

📌 S — Scenario 2 (Banking, Non-Technical)

Nord Bank-এর একটা Branch-এ একটা Network Device-এর Setting কেউ Manually পরিবর্তন করেছিল, কিন্তু আগের Setting কোথাও Save করে রাখা হয়নি। পরে যখন নতুন Setting-এ সমস্যা দেখা দেয়, তখন কেউ মনে করতে পারছিল না আগে ঠিক কী Setting ছিল।

🎯 T — Task

আজকের লক্ষ্য:
- Configuration Backup বলতে কী বোঝায় তা বোঝা
- কেন নিয়মিত এবং স্বয়ংক্রিয়ভাবে Backup নেওয়া Manual Backup-এর চেয়ে ভালো
- একটা Backup-এ ন্যূনতম কী তথ্য থাকা উচিত তার ধারণা পাওয়া

👀 O — Output

Module শেষে learner বলতে পারবে:
- Config Backup কেন গুরুত্বপূর্ণ
- Manual Backup-এর সাধারণ সমস্যা কী (ভুলে যাওয়া, Inconsistent হওয়া)
- একটা Automated Backup Process-এর মূল ধাপ কী কী হতে পারে (Connect → Retrieve → Save)

🤔 R — Reason

Setting বা Configuration সময়ের সাথে পরিবর্তন হয়, আর মানুষ ভুলে যায় বা ব্যস্ত থাকে। যদি Backup নেওয়ার কাজটা Automation-এর উপর ছেড়ে দেওয়া হয়, তাহলে এটা নিয়মিত এবং একইভাবে (Consistent) হয়, মানুষের মনে রাখার উপর নির্ভর করতে হয় না।

🧠 Memory Tip

Config Backup-কে ভাবা যেতে পারে একটা "Save Game" Feature-এর মতো — খেলার মাঝে কিছু ভুল হলে আগের Save Point-এ ফিরে যাওয়া যায়, ঠিক তেমনি Device-এর Setting ভুল হলে আগের Backup-এ ফিরে যাওয়া যায়।

⚠️ L — Limitation

- Backup নেওয়া মানেই এই না যে সেটা সহজে Restore করা যাবে — Restore Process আলাদাভাবে Test না করলে Backup আসলে কাজ করবে কিনা তা নিশ্চিত না
- সব Backup একই জায়গায় রাখলে, সেই জায়গা নষ্ট হলে সব Backup একসাথে হারিয়ে যেতে পারে
- Backup নেওয়ার সময় যদি Device Busy বা Unreachable থাকে, তাহলে সেই দিনের Backup miss হয়ে যেতে পারে
- এই ধারণা এখনও কীভাবে Backup সংরক্ষণ করা হবে বা কতদিন রাখা হবে তা নিয়ে কিছু বলে না

✋ Y — Your Turn

একটা Device বা System চিন্তা করে লিখতে হবে যার Setting মাঝে মাঝে পরিবর্তন হয়, এবং সেটার জন্য কেন Regular Backup দরকার তা ২-৩ বাক্যে ব্যাখ্যা করতে হবে।

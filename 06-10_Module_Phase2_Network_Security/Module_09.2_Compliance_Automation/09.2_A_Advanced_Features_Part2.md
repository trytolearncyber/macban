📘 Module 09.2 — Advanced Features Part 2 (Section A)

📌 S — Scenario 1 (Non-Banking)

একজন Blogger-এর কাছে অনেক পুরনো Article আছে, যেগুলোর Word Count, Reading Time, আর Tag আলাদাভাবে বের করা দরকার। Visual Node দিয়ে এই ধরনের Custom হিসাব করা কঠিন, কারণ এটা একটা নির্দিষ্ট Logic মেনে হিসাব করতে হয়, যা আগে থেকে বানানো কোনো Node দিয়ে সরাসরি সম্ভব না।

📌 S — Scenario 2 (Banking, Non-Technical)

Nord Bank-এর একটি Automation-এ Customer-এর তথ্য দেখে ঠিক করতে হবে তিনি কোন Service-এর জন্য যোগ্য। যেমন Account-এর ধরন, Account Balance, Customer Category, এবং Relationship-এর সময়কাল একসাথে বিবেচনা করে সিদ্ধান্ত নিতে হবে। এই ধরনের সিদ্ধান্ত আগে থেকে তৈরি সাধারণ Node দিয়ে সবসময় করা যায় না, কারণ প্রতিটি Bank-এর নিজস্ব Business Rule থাকে।

📌 S — Scenario 3 (Banki_Tech)

Nord Bank-এর একটা Automation-এ একটা জটিল হিসাব করা দরকার — যেমন একটা Transaction-এর তথ্য থেকে সেটা কোন ধরনের Rule ভঙ্গ করছে কিনা তা নির্ণয় করা। এই ধরনের Custom Logic আগে থেকে বানানো কোনো সাধারণ Node দিয়ে করা সম্ভব না, কারণ প্রতিটা Bank-এর নিয়ম আলাদা।

🎯 T — Task

আজকের লক্ষ্য:
- Code Node বলতে কী বোঝায় তা বোঝা — যখন সাধারণ Node যথেষ্ট না, তখন কী করা হয়
- Custom Logic বলতে কী বোঝায় তার ধারণা পাওয়া
- কখন সাধারণ Visual Node যথেষ্ট এবং কখন Code দরকার হয়, তার পার্থক্য চেনা

👀 O — Output

Module শেষে learner বলতে পারবে:
- Code Node কী এবং এটা সাধারণ Node থেকে কীভাবে আলাদা
- কোন ধরনের কাজে Custom Logic দরকার হতে পারে (যেমন জটিল হিসাব, নির্দিষ্ট শর্ত পরীক্ষা করা)
- Code ব্যবহার করলে কী সুবিধা পাওয়া যায় (Flexibility) এবং কী চ্যালেঞ্জ থাকে (জটিলতা)

🤔 R — Reason

Visual Node-গুলো সাধারণ, বহুল ব্যবহৃত কাজের জন্য বানানো হয়, কিন্তু প্রতিটা প্রতিষ্ঠানের নিজস্ব কিছু নিয়ম বা হিসাব থাকে যা আগে থেকে বানানো Node দিয়ে সম্ভব না। এই ধরনের ক্ষেত্রে একটা ছোট Code লিখে সেই নির্দিষ্ট Logic Automation-এর মধ্যে যোগ করা যায়, যা Automation-কে আরও Flexible করে তোলে।

🧠 Memory Tip

Code Node-কে ভাবা যেতে পারে একটা "Custom-Made Tool"-এর মতো — দোকানে পাওয়া সাধারণ Tool দিয়ে বেশিরভাগ কাজ হয়, কিন্তু কোনো বিশেষ কাজের জন্য নিজে বানানো একটা Tool দরকার হয়।

⚠️ L — Limitation

- Code Node ব্যবহার করতে হলে Logic নিজে সঠিকভাবে চিন্তা করে লিখতে হয় — ভুল Logic লিখলে Automation ভুল ফলাফল দেবে, কিন্তু সেটা বোঝা সবসময় সহজ না
- যারা Code পড়তে বা লিখতে পারে না, তাদের জন্য Code Node বোঝা এবং পরে Maintain করা কঠিন হয়ে যায়
- একটা Automation-এ অনেক বেশি Code Node ব্যবহার করলে, পুরো Workflow বোঝা এবং Debug করা জটিল হয়ে যায়
- এই ধারণা এখনও নির্দিষ্ট কোনো Programming Language বা Syntax নিয়ে কিছু বলে না — এটা শুধু "কখন এবং কেন" তা বোঝানোর জন্য

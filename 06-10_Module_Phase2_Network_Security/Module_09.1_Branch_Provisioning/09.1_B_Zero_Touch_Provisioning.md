📘 Module 09.1 — Zero-Touch Branch Provisioning (Section B)

📌 S — Scenario (Nord Bank / Banking-Specific)

Nord Bank যখন একটা নতুন Branch খোলে, তখন Network Engineer-কে সেই Branch-এ গিয়ে বা Remote-এ Manually Router Config করতে হয়, FortiGate Setup করতে হয়, এবং DNS/DHCP ঠিক করতে হয়। এই পুরো কাজে ৪-৫ দিন লেগে যায়, এবং প্রতিটা Branch-এর জন্য একই ধরনের কাজ Manually বারবার করতে হয়, যেখানে ভুল হওয়ার সম্ভাবনাও থাকে।

🚨 Challenge

- প্রতিটা নতুন Branch Setup করতে অনেক সময় লাগছিল (৪-৫ দিন)
- Manual Configuration-এ Human Error-এর ঝুঁকি বেশি — যেমন ভুল IP Range বসিয়ে দেওয়া
- Branch সংখ্যা বাড়লে (900+ Branch) এই Manual Process Scale করা কঠিন হয়ে যাচ্ছিল
- একেক Engineer একেকভাবে Configure করলে Branch-গুলোর মধ্যে Consistency থাকছিল না

✅ Solution

একটা Zero-Touch Provisioning Workflow তৈরি করতে হবে, যেখানে একটা Webhook Request-এর মাধ্যমে নতুন Branch-এর তথ্য (নাম, Location, IP Range) দিলেই Automation নিজে থেকে Router, FortiGate, এবং DNS/DHCP-এর জন্য প্রয়োজনীয় Config তৈরি করে সংশ্লিষ্ট Device-এ Push করে দেবে।

🎯 T — Task

Nord Bank-এর জন্য একটা Zero-Touch Branch Provisioning Architecture ডিজাইন করতে হবে, যা Webhook দিয়ে শুরু হয়ে Config Generation এবং Device-এ Push পর্যন্ত পুরো Process Automate করবে।

👀 O — Output

একটা Zero-Touch Provisioning Architecture Document যেখানে থাকবে:
- Webhook Input Design — নতুন Branch Request-এ কী কী তথ্য থাকবে
- Config Generation Logic — Branch Name, Location, IP Range থেকে কীভাবে Device-Specific Config তৈরি হবে
- Device Push Strategy — Cisco Router, FortiGate, DNS/DHCP Server-এ কীভাবে Config পাঠানো হবে
- Validation Step — Push করার আগে বা পরে Config সঠিক কিনা তা কীভাবে যাচাই হবে

🤔 R — Reason

Branch সংখ্যা বাড়ার সাথে সাথে Manual Provisioning Process আর Scale করা যায় না — সময় এবং Human Error দুটোই বেড়ে যায়। Zero-Touch Provisioning ব্যবহার করলে প্রতিটা Branch একইভাবে, দ্রুত, এবং কম ভুলে Setup হয়, যা Consistency এবং Speed দুটোই নিশ্চিত করে।

🏦 Real-World Use Case

Nord Bank একটা Webhook তৈরি করে যেখানে Branch Manager একটা Form পূরণ করলে (Branch Name, Location, IP Range) সেটা Automation-এ যায়। Automation সেই তথ্য থেকে Cisco এবং FortiGate-এর জন্য আলাদা Config Template তৈরি করে, তারপর SSH-এর মাধ্যমে সংশ্লিষ্ট Device-এ Push করে। Branch Setup-এর সময় ৪-৫ দিন থেকে কমে ১ দিনে নেমে আসে।

🧠 Memory Tip

Zero-Touch Provisioning-কে ভাবা যেতে পারে একটা "Franchise Restaurant Opening"-এর মতো — প্রতিটা নতুন Branch একই Recipe, একই Layout, একই Standard মেনে খোলে, যাতে কোনো Branch আলাদাভাবে ভুল Setup না হয়।

⚠️ L — Limitation

- Config Template যদি ভুলভাবে তৈরি করা হয়, তাহলে সেই ভুল Template দিয়ে তৈরি হওয়া প্রতিটা Branch-এই একই ভুল ছড়িয়ে যাবে — Manual-এর চেয়ে ভুলের প্রভাব বড় আকারে হতে পারে
- প্রতিটা Branch-এর Physical Environment (যেমন ISP Availability, Local Network Condition) আলাদা হতে পারে, যা একটা Generic Template সবসময় Handle করতে পারে না
- Zero-Touch মানে এই না যে Verification-ও দরকার নেই — Push করার পর Device সত্যিই সঠিকভাবে কাজ করছে কিনা তা যাচাই করা এখনও জরুরি
- Webhook-এ ভুল বা Unauthorized তথ্য দিয়ে Request পাঠানো হলে (Validation না থাকলে) Automation ভুল Branch-এর জন্য Config Push করে দিতে পারে — তাই Input Validation এবং Access Control আবশ্যক

✋ Y — Your Turn

Nord Bank-এর একটা নতুন Branch-এর জন্য Zero-Touch Provisioning Webhook-এ কী কী তথ্য অবশ্যই থাকা উচিত এবং কোন তথ্য ভুল হলে সবচেয়ে বড় সমস্যা হতে পারে তা ২-৩ বাক্যে লিখতে হবে।

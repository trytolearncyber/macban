📘 Module 09.3 — Advanced Features Part 3 (Section A)

📌 S — Scenario 1 (Non-Banking)

একজন Retail Shop Owner-এর কাছে Sales Data আসে একবার CSV File আকারে, আরেকবার একটা App থেকে JSON আকারে। দুটো Format আলাদা হওয়ায় একসাথে রেখে হিসাব করা কঠিন হয়ে যায়। এছাড়া, সে জানতে চায় গত মাসে মোট কত টাকার Sale হয়েছে এবং প্রতিদিন গড়ে কতজন Customer এসেছে — কিন্তু আলাদা আলাদা তথ্য থেকে এই Summary বের করা Manually সময়সাপেক্ষ।

📌 S — Scenario 2 (Banking, Non-Technical)

Nord Bank-এর কাছে বিভিন্ন Branch-এর Device তথ্য আসে ভিন্ন ভিন্ন Format-এ — কোনো Device থেকে XML, কোনোটা থেকে JSON। এই তথ্যগুলোকে এক জায়গায় এনে বলা দরকার মোট কতগুলো Device Online আছে এবং Average Bandwidth Usage কত — কিন্তু ভিন্ন Format-এর কারণে এই Summary বের করা কঠিন।

🎯 T — Task

আজকের লক্ষ্য:
- Data Transformation বলতে কী বোঝায় — এক Format-এর তথ্যকে আরেক Format-এ রূপান্তরের ধারণা
- Data Aggregation বলতে কী বোঝায় — অনেক তথ্য থেকে একটা Summary (Total, Average, Count) বের করার ধারণা
- কেন ভিন্ন Format-এর তথ্য একসাথে কাজ করার আগে একই Format-এ আনা দরকার

👀 O — Output

Module শেষে learner বলতে পারবে:
- Data Transformation কী এবং কেন এটা দরকার হয়
- Aggregation-এর সাধারণ ধরন কী কী (Sum, Average, Count)
- ভিন্ন Source থেকে আসা তথ্য একসাথে Analyze করার আগে কী প্রস্তুতি দরকার

🤔 R — Reason

বিভিন্ন System ভিন্ন ভিন্ন Format-এ তথ্য দেয়, কিন্তু একটা Automation-কে সিদ্ধান্ত নিতে হলে বা Summary বানাতে হলে সব তথ্য একই Format-এ থাকা দরকার। তাছাড়া, একটার পর একটা আলাদা তথ্য দেখে সিদ্ধান্ত নেওয়ার চেয়ে একটা Summary (যেমন Total, Average) দেখে সিদ্ধান্ত নেওয়া অনেক দ্রুত এবং সহজ।

🧠 Memory Tip

Data Transformation = "বিভিন্ন Currency-কে একই Currency-তে Convert করা, তারপর যোগ করা"
Data Aggregation = "অনেকগুলো আলাদা Receipt থেকে একটা Total Bill বের করা"

⚠️ L — Limitation

- Data Transformation ভুলভাবে করলে (যেমন ভুল Field Map করা) ভুল তথ্য নিয়ে হিসাব হয়ে যাবে, যা পরে ধরা কঠিন
- Aggregation করলে বিস্তারিত তথ্য (Individual Detail) হারিয়ে যায় — শুধু Summary দেখে কখনো কখনো গুরুত্বপূর্ণ একটা Outlier বা ব্যতিক্রম চোখ এড়িয়ে যেতে পারে
- ভিন্ন Source-এর তথ্য একসাথে আনার আগে বোঝা দরকার প্রতিটা Source আসলে কী পরিমাপ করছে — একই নাম থাকলেও দুই Source ভিন্ন জিনিস বোঝাতে পারে
- এই ধারণা এখনও কোনো নির্দিষ্ট Tool বা Syntax দিয়ে কীভাবে Transform/Aggregate করা হয় তা বলে না

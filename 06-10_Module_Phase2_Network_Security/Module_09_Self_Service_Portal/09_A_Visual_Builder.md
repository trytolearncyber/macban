📘 Module 09 — Introduction to n8n's Visual Builder
📌 S — Scenario
Nord Bank-এ একজন নতুন IT Support Staff Join করেছে, যার Coding Experience নেই। তাকে একটি Simple Automation Task (যেমন Daily Report Email) তৈরি করতে বলা হয়েছে, কিন্তু সে Code লিখতে জানে না।
🚨 Challenge

Traditional Automation-এ Coding Skill (Python, JavaScript) প্রয়োজন হয়
Non-Technical Staff-দের জন্য Automation তৈরি করা কঠিন হয়ে যায়
একটি ছোট পরিবর্তনের জন্যও Developer-এর সাহায্য নিতে হয়, যা সময়সাপেক্ষ

✅ Solution
n8n-এর Visual Builder ব্যবহার করে Code না লিখেই, শুধু Drag & Drop করে Automation Workflow তৈরি করা যায়।
🎯 T — Task
আজকের Learning Objective হলো n8n-এর Visual Builder Interface বুঝা।
ছোট ছোট ধাপ:

Visual Builder-এর তিনটি প্রধান অংশ চেনা
Node কী তা বুঝা
Node-গুলো কীভাবে একসাথে Connect হয় তা বুঝা

👀 O — Output
এই Module শেষে, একজন Learner বুঝতে পারবে:
Interface Part→PurposeNode Palette (Left Side)→সব Available Node-এর তালিকা এখানে থাকেCanvas (Center)→যেখানে Node বসিয়ে Workflow তৈরি করা হয়Properties (Right Side)→নির্বাচিত Node-এর Setting এখানে Configure করা হয়
🤔 R — Reason
Visual Builder ব্যবহার করার কারণ:

Non-Technical Staff-ও নিজে থেকে Automation তৈরি করতে পারে
Workflow দেখতে সহজ (Visual), তাই Understand ও Debug করা সহজ হয়
Developer-এর উপর নির্ভরতা কমে, প্রতিটি ছোট কাজের জন্য অপেক্ষা করতে হয় না

📊 Simple Diagram
[Node Palette] → Drag → [Canvas: Node1] → Connect → [Canvas: Node2]
                              (Properties Panel দিয়ে প্রতিটি Node Configure হয়)
🏦 Real-World Use Case
Nord Bank-এর NOC Team-কে প্রতিদিন ৫০০+ Device-এর Status manually check করতে হয়। এই কাজে ৪ ঘন্টা সময় লাগে। n8n Workflow ব্যবহার করলে এই কাজ ৫ মিনিটে করা সম্ভব — এবং Visual Builder-এর কারণে এটি তৈরি করতে Coding Background প্রয়োজন হয় না।
🧠 Memory Tip
Palette → Canvas → Properties — বাম থেকে ডানে এই তিনটি অংশ মনে রাখলেই n8n Interface বোঝা সহজ হয়ে যায়।
